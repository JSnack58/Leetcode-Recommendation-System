# Documentation Index

| Directory | Purpose |
|---|---|
| [planning/](planning/) | Active planning area — roadmap, sprint notes, experiment log |
| [architecture/](architecture/) | System design documents: data flow, component map, tier logic |
| [adr/](adr/) | Architecture Decision Records (MADR format) |

## How to navigate

- **Starting a new feature?** Check [planning/roadmap.md](planning/roadmap.md) for the current phase and [adr/](adr/) for decisions that may constrain your approach.
- **Unsure why something was built a certain way?** Browse the ADRs — each records the context, alternatives considered, and the chosen option.
- **Changing the system design?** Update the relevant doc in [architecture/](architecture/) and write a new ADR if the change is significant.
